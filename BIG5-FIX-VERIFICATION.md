# Big5 編碼偵測修正驗證報告

## 測試日期
2026-01-28

## 問題描述
用戶反映 Big5 編碼的檔案被誤判為 GBK，導致顯示「已成功轉換為 UTF-8 (原編碼: gbk)」而非正確的「原編碼: big5」。

## 測試檔案
- **檔案路徑**: `c:\C_chwa\code\ch06\ch6-1-1.c`
- **實際編碼**: Big5
- **檔案大小**: 256 bytes

## 診斷結果

### 字節分析
```

0000: 23 69 6e 63 6c 75 64 65 20 3c 73 74 64 69 6f 2e  | #include <stdio.
...

0040: e9 a4 4a ae f0 b7 c5 20 3d 3e 22 29 3b 0d 0a 20  | ..J.... =>");..
...
0080: 70 72 69 6e 74 66 28 22 a5 5b a5 f3 a5 7e ae 4d  | printf(".[...~.M
```

### Big5 特徵識別
發現 6 個 Big5 特有特徵（第二字節在 0x40-0x7E 範圍）：
- `a4 4a` (入)
- `a5 5b` (加)
- `a5 7e` (件)
- `ae 4d` (套)
- `a4 66` (天)
- `b8 6e` (罩)

### 分數統計
- ASCII 字符數: 229
- Big5 雙字節數: 24
- **Big5 特有特徵數: 6** (權重 x2)
- GBK 雙字節數: 24
- **GBK 特有特徵數: 0** (權重 x2)

### 判斷結果
```
Big5 總分: 36 (24 + 6*2)
GBK 總分: 24

✓ 最終判斷: Big5 (因為 big5SpecificScore = 6 > 3)
```

## 改進的判斷邏輯

### 關鍵改進
```typescript
// Big5 特有的第二字節範圍 (0x40-0x7E)
if (byte2 >= 0x40 && byte2 <= 0x7E && byte2 !== 0x7F) {
    big5SpecificScore += 2;  // 給予更高權重
}
```

### 判斷條件
```typescript
if (big5SpecificScore > 3 || (totalBig5 > gbkScore && big5Score > 3)) {
    return 'big5';  // Big5 特徵數 > 3 就判定為 Big5
}
```
