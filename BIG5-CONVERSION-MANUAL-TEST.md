# Big5 轉換功能手動測試清單

## 📋 測試目的
驗證 Big5 轉換功能的單向寫入行為，確保檔案正確儲存為 ANSI Big5 編碼，並且用戶體驗流暢。

## 🔧 測試環境準備

1. 確保已安裝 `iconv-lite` 套件：
   ```bash
   npm install
   ```

2. 準備測試檔案：
   - 建立一個包含中文註解的 C/C++ 檔案
   - 內容範例：
     ```c
     // 這是繁體中文測試
     #include <stdio.h>
     
     int main() {
         printf("Hello 世界\n");
         return 0;
     }
     ```

## ✅ 測試案例

### 測試 1：基本轉換流程
**步驟：**
1. 在 VS Code 中開啟測試檔案（UTF-8 編碼）
2. 執行命令：`C/C++ Smart Runner: 轉換為 Big5`
3. 觀察第一個警告對話框
4. 點擊「確定轉換」
5. 觀察成功訊息

**預期結果：**
- ✅ 顯示警告對話框，說明轉換後 VS Code 無法顯示
- ✅ 檔案儲存為 Big5 編碼
- ✅ 編輯器自動關閉該檔案
- ✅ 顯示成功訊息，包含使用 Dev-C++ 開啟的建議
- ✅ 在 Dev-C++ 或支援 Big5 的編輯器中開啟，中文顯示正常

### 測試 2：取消第一次警告
**步驟：**
1. 在 VS Code 中開啟測試檔案
2. 執行命令：`C/C++ Smart Runner: 轉換為 Big5`
3. 在第一個警告對話框點擊「取消」

**預期結果：**
- ✅ 檔案保持開啟
- ✅ 檔案內容未修改
- ✅ 檔案編碼仍為 UTF-8
- ✅ 無錯誤訊息

### 測試 3：未儲存修改的處理
**步驟：**
1. 在 VS Code 中開啟測試檔案
2. 修改檔案內容（不要儲存）
3. 執行命令：`C/C++ Smart Runner: 轉換為 Big5`
4. 觀察警告對話框
5. 點擊「儲存並轉換」
6. 在下一個確認對話框點擊「確定轉換」

**預期結果：**
- ✅ 顯示「檔案有未儲存的修改」警告
- ✅ 提供「儲存並轉換」和「取消」選項
- ✅ 選擇「儲存並轉換」後，檔案自動儲存
- ✅ 繼續顯示轉換確認對話框
- ✅ 最終成功轉換為 Big5

### 測試 4：未儲存修改時取消
**步驟：**
1. 在 VS Code 中開啟測試檔案
2. 修改檔案內容（不要儲存）
3. 執行命令：`C/C++ Smart Runner: 轉換為 Big5`
4. 在「儲存並轉換」對話框點擊「取消」

**預期結果：**
- ✅ 檔案保持開啟
- ✅ 未儲存的修改仍然存在
- ✅ 檔案編碼仍為 UTF-8
- ✅ 無轉換操作

### 測試 5：Big5 檔案驗證
**步驟：**
1. 執行測試 1，將檔案轉換為 Big5
2. 用命令行工具檢查檔案編碼：
   ```powershell
   # Windows PowerShell
   Get-Content test.c -Encoding Default
   ```
3. 在 Dev-C++ 中開啟檔案
4. 在記事本中開啟（編碼選擇 ANSI）

**預期結果：**
- ✅ 檔案確實以 Big5 編碼儲存（非 UTF-8）
- ✅ Dev-C++ 中中文正常顯示
- ✅ 記事本以 ANSI 編碼開啟，中文正常
- ✅ 直接在 VS Code 重新開啟該檔案會顯示亂碼（預期行為）

### 測試 6：轉換後的逆向操作
**步驟：**
1. 執行測試 1，將檔案轉換為 Big5
2. 在 VS Code 中重新開啟該 Big5 檔案（會顯示亂碼）
3. 執行命令：`C/C++ Smart Runner: 從 Big5 轉換為 UTF-8`
4. 檢查檔案內容

**預期結果：**
- ✅ 重新開啟時顯示為亂碼（預期行為）
- ✅ 執行「從 Big5 轉換為 UTF-8」後，亂碼變回正常中文
- ✅ 檔案可以在 VS Code 中正常編輯
- ✅ 按 Ctrl+Z 可以撤銷轉換

### 測試 7：對話框文字內容
**步驟：**
1. 執行轉換命令
2. 仔細閱讀所有對話框的文字

**預期結果：**
對話框應包含以下關鍵資訊：
- ✅ 警告：轉換後 VS Code 無法正確顯示
- ✅ 說明：檔案將儲存為 ANSI Big5 編碼
- ✅ 說明：檔案會自動關閉
- ✅ 建議：使用 Dev-C++ 開啟
- ✅ 提示：如何轉回 UTF-8
- ✅ 成功訊息清楚明確

### 測試 8：錯誤處理（模擬）
**注意：** 此測試需要特殊環境設置，可選做

**步驟：**
1. 暫時重命名 `node_modules/iconv-lite` 目錄
2. 重啟 VS Code
3. 嘗試執行轉換命令
4. 恢復 `node_modules/iconv-lite` 目錄

**預期結果：**
- ✅ 顯示錯誤訊息：「未安裝 iconv-lite 套件」
- ✅ 提供安裝指令提示
- ✅ 檔案保持開啟，未修改
