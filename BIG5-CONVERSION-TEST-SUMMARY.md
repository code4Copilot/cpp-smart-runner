# Big5 轉換功能測試總結

## 📅 更新日期：2026年1月29日

## 🎯 功能改進說明

將 Big5 轉換功能從「轉換後重新載入（導致亂碼）」改為「單向寫入並關閉檔案」的方案。

### 改進前的問題
- 轉換為 Big5 後，VS Code 以 UTF-8 解讀 Big5 字節 → 顯示亂碼
- 使用者無法繼續編輯，體驗不佳

### 改進後的行為
- 轉換前雙重確認，明確告知後果
- 寫入 ANSI Big5 格式
- 自動關閉編輯器，避免亂碼顯示
- 清楚說明如何使用（Dev-C++）和如何轉回（UTF-8 轉換）

## 🎓 測試重點說明

### 為什麼需要 Mock 對話框？

VS Code 的 `showWarningMessage` 會暫停執行等待用戶點擊，在自動化測試中：
- 無法真正顯示對話框（headless 環境）
- 需要模擬用戶的選擇
- 使用 Sinon 的 `stub` 來攔截並回傳預設值

### 為什麼需要手動測試？

某些方面自動化測試無法完全涵蓋：
- 對話框文字的清晰度（需要人工閱讀）
- 與外部工具的互操作性（Dev-C++、記事本）
- 實際使用流程的順暢度
- 視覺呈現和用戶體驗
